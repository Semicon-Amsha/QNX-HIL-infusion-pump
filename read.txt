The QNX image has a read-only filesystem so we can't permanently save anything. Every reboot wipes /var. So we:

Set a static IP on the Pi's ethernet
Created SSH keys on your PC
Put your public key in /var/.ssh/ (writable RAM disk)
Started sshd with a custom config pointing to that key
SSHed in without a password


Every reboot — run this ONE script on PuTTY serial
Copy this entire block and right-click paste into PuTTY after every reboot:
ifconfig genet0 192.168.1.50 netmask 255.255.255.0 && route add default 192.168.1.1 && mkdir -p /var/.ssh && chmod 700 /var/.ssh && echo "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPlZLgrbMQe91VmT0E0deQdNDaiK39QlAgp5KkQTztkX shukl@pegasus" > /var/.ssh/authorized_keys && chmod 600 /var/.ssh/authorized_keys && cat /etc/ssh/sshd_config > /var/sshd_final && echo "AuthorizedKeysFile /var/.ssh/authorized_keys" >> /var/sshd_final && echo "PermitRootLogin yes" >> /var/sshd_final && echo "StrictModes no" >> /var/sshd_final && slay sshd ; /usr/sbin/sshd -f /var/sshd_final -o "AuthorizedKeysFile /var/.ssh/authorized_keys" && dhcpcd -f /var/dhcpcd_static.conf
Then from Windows CMD:
ssh-keygen -R 192.168.1.50
ssh -i %USERPROFILE%\.ssh\id_ed25519 root@192.168.1.50
